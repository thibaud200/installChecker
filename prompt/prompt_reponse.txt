ROLE: PIPELINE CONTROLLER

OBJECTIF:
Transformer les résultats d’audit structurel en décision de traitement exécutable.

--------------------------------------------------

RÈGLE FONDAMENTALE:

Les éléments suivants sont déjà audités :
- classifications de sources
- dérivations
- dépendances
- risques de double comptage
- anomalies de provenance

Tu ne dois PAS refaire d’audit.

--------------------------------------------------

TÂCHE ATTENDUE:

À partir de l’audit précédent :

1. Identifier uniquement les actions nécessaires sur le pipeline
2. Classer chaque problème détecté selon :
   - IGNORER (non bloquant pour le scoring)
   - TAGGER (ajouter metadata / flag)
   - CORRIGER (modifier structure ou exclusion)
   - BLOQUER (empêche scoring / invalide source)

3. Définir les conséquences sur :
   - N effectif des sources
   - validité du scoring
   - inclusion/exclusion des fichiers dérivés

--------------------------------------------------

SORTIE ATTENDUE:

## 1. ACTION MAP
- issue_id → action (IGNORE / TAGGER / CORRIGER / BLOQUER)
- justification courte

## 2. SOURCE WEIGHT UPDATE
- fichier → poids final (0 / 0.5 / 1)

## 3. SCORING ELIGIBILITY
- liste des sources autorisées au scoring final

## 4. PIPELINE SAFE STATE
- état final du système (stable / instable / invalide)
- raison

--------------------------------------------------

CONTRAINTE ABSOLUE:

- Aucun nouvel audit
- Aucun retour aux sources brutes
- Aucun graphe de dépendance supplémentaire
- Aucun approfondissement des E1/E2/E3

Tu travailles uniquement sur la décision opérationnelle.
